# 速度王 SpeedStar ⚡️

一個給小朋友玩的「測速度」網頁 App，三個模式：

| 模式 | 玩法 | 偵測方式 |
|------|------|----------|
| 🏃 跑步 | 橫向跑過畫面，量速度 | 鏡頭 + AI 骨架追蹤（MoveNet） |
| 🚩 比賽 | 兩人輪流跑同一段，比贏家 | 同上，記兩次成績 + 動畫重播 |
| ⚽️ 踢球 | 球穿過兩條線，量球速 | 鏡頭 + 光閘偵測（只看線上的移動） |

純前端網頁，免安裝、通用版（不綁特定名字）。可加到手機主畫面像真 App。

**比賽選手名字可自訂**：比賽模式點名字就能改成你們自己的名字，會自動記住。

**事後重播 + 存手機（可選特效）**：因為小孩在跑、看不到螢幕，每次測速會自動把鏡頭畫面錄下來，測完可在 App 裡重播，並提供兩種存檔：

- **✨ 存特效版**：把 AI 骨架、起終點線、速度數字做進影片（事後背景合成，需幾秒）。
- **💾 存原始版**：乾淨的鏡頭畫面。

兩個都能存。按下會叫出系統分享/儲存（可存進相簿）；不支援分享的瀏覽器會改成直接下載。

**比賽模式**除了各自存兩人的影片，還可按「🏁 存並排對比影片」把兩段影片**左右並排**成一支對比影片（各自帶骨架/速度，贏的人加 👑）。

**踢球用「光閘」偵測**：只盯著起點線、終點線兩條窄帶上有沒有東西通過，球穿過第一條→第二條的時間就是球速。這樣旁邊有人在動也不會被干擾（比抓「整個畫面的移動中心」穩很多）。把兩條線拖到**球會經過、人不會擋到**的位置最準；測不到就把「靈敏度」調高。

為什麼分兩階段：即時錄影只錄乾淨串流（很輕量、不拖慢追蹤），同時把每格的疊圖資料記成很小的數字；等小孩跑完才在背景把特效合成上去，所以偵測完全不受影響。可用面板「📹 自動錄影」開關關閉錄影。需要較新的手機瀏覽器（iPhone 建議 iOS 17 以上）。

## 怎麼玩（重要）

1. 手機**架在側面**（用腳架或靠著東西），鏡頭對準跑道/球的路線。
2. 讓小朋友**橫向**通過畫面（不要朝鏡頭跑來）。
3. 用手指**拖曳綠線（起點）和紅線（終點）**對準實際的起跑點與終點。
4. 把面板上的「距離」調成兩條線之間的**實際公尺數**（量一下地面距離）。
5. 按開始 → 倒數 → 通過兩條線就會顯示速度！

> ⚠️ 這是「趣味估算」：手機只有一般 30 格/秒，球太快會糊。準度靠側面拍、背景單純、距離量準。沒鏡頭時可用「手動計時」。

## 在電腦上測試（開發用）

鏡頭需要安全環境（`localhost` 或 `https`），直接雙擊 HTML 開不了鏡頭。請用本機伺服器：

```powershell
# 方法 A：Python
python -m http.server 8000

# 方法 B：Node
npx serve .
```

然後在 **Chrome** 打開 http://localhost:8000 （桌機用 webcam 也能測）。

---

## 放到手機上玩（需要 HTTPS）

手機鏡頭一定要 `https`。最簡單的兩種免費做法：

- **Netlify Drop**：到 https://app.netlify.com/drop 把整個資料夾拖進去，馬上得到一個 https 網址。
- **GitHub Pages / Vercel**：上傳這個資料夾即可。

拿到網址後用手機瀏覽器打開 → 選單「加到主畫面」→ 就變成 App 圖示。

## 檔案

- `index.html` — 介面骨架
- `style.css` — 樣式（深色、大數字、三模式配色）
- `vision.js` — 鏡頭擷取、AI 骨架、移動追蹤
- `app.js` — 三模式邏輯、計時門、紀錄、語音鼓勵、慶祝動畫
- `manifest.webmanifest` — 加到主畫面用
